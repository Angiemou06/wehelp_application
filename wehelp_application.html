兩百字內的個人簡介。

我是牟彥蓁，今年28歲，畢業於臺灣大學土木工程學系，現職為水利工程師，也兼職幾份家教工作。
工作了幾年，發現自己喜歡在和他人討論的過程中，將複雜的問題拆解、條理化所帶來的成就感。
喜歡職場氛圍友善自由、少些繁文縟節，也喜歡分享知識、陪伴學生成長的成就感。
為了更理想職場環境，並兼顧喜歡的工作，擁有更平衡快樂的人生，而走上轉職軟體工程師的道路。
我自認不是特別聰明的人，但是一個願意為了目標而持續努力不懈的人。

為了成為軟體工程師，曾做過什麼努力？

  • 時常瀏覽軟體工程師相關職缺資訊，了解軟體工程師職缺要求具備的專業技能及目前就業市場動態。
  • 主動利用Youtube及網路文章等網路資源學習程式語言、解決學習程式語言時遇到的困難。
  • 報名臺大資工系統訓練班課程，學習「Python程式設計」、「JavaScript網頁前端工程入門」、「C/C++基礎程式設計」、「資料結構與C++進階班」，及「Node.js 和 MongoDB 網站後端開發」等課程。其中，「Python程式設計」、「JavaScript網頁前端工程入門」、「C/C++基礎程式設計」三門課程已完成修習。
  • 開始使用Python刷leetcode。
  • 以「W3schools」網站進行HTML、CSS，及JavaScript語法練習。
  • 專案介紹

如果參與這個計畫，會怎麼安排學習時間？

在課程正式開始前，我會進行以下準備：

  1. 完成臺大資工系統訓練班「資料結構與C++進階班」及「Node.js 和 MongoDB 網站後端開發」課程。
  2. 持續使用Python進行leetcode刷題。
  3. 持續使用「W3schools」複習HTML、CSS、JavaScript語法。
  4. 利用網路資源自學Python Flask及MySQL等相關技術。

我預計在第一階段八週的課程中，以在職的方式投入課程，由於第一階段預計以在職方式投入課程，所以我會在課程開始前，先透過彭彭的影片自學Python Flask網站後端開發、MySQL資料庫，及Vue框架等課程，並藉由台大資工系統訓練班「Node.js 和 MongoDB 網站後端開發」課程，學習Node.js 和 MongoDB相關知識，避免課程開始後，因為在職投入時間較少而導致進度落後。
第一階段的課程正式開始後，我預計利用平日早上5點至7點、晚上7點至12點的時間投入課程，假日則預計投入早上9點至下午7點的時間進行學習，中午12點至1點為休息時間，預估每周的學習時數約為51小時，我了解每週一的線上會議將於早上十點舉行，會錄影紀錄，我會爭取在這個時段請特休假，加入會議了解當週的作業安排及關鍵技術。
第二階段起的課程，我預計辭去正職工作投入轉職，從過往學習程式語言的經驗裡，我自認自己不是屬於反應特別快的人，在coding過程中，從釐清問題、拆解問題、找尋答案，到成功解決問題，都要花費不少時間，所以我預估全職轉職期間，每周約投入60個小時的時間進行學習，每天預計從早上9點開始學習至下午6點，中午12點至下午1點休息一小時。

是否有想要加入的軟體公司？為什麼想加入該公司？

我轉職的最終目標是想進去外商公司，如：Houzz、Microsoft這類公司擔任軟體工程師，或是找外商的remote職缺，主要是因為外商公司普遍薪水和福利較好。
但轉職初期，可預期非本科系又沒有相關工作經驗的我，要獲得前述外商職缺的機會較小，因此我沒有設限特別想加入的軟體公司。
但我希望能加入工作氣氛輕鬆活潑、同事長官間相處融洽、準時上下班的公司，因為這樣的工作環境是我非常嚮往的，希望能透過轉職而獲得工作與生活的平衡。

請描述一件讓你產生明顯負面情緒的事情，你如何處理該情緒？

曾經讓我產生明顯負面情緒的事情之一，是撰寫碩士班畢業論文時，當時我選的論文題目是要結合程式語言(MATLAB)和數值模擬方法，撰寫一個二維水理及輸砂數值模式，將該模式運用於模擬均勻混合河口泥砂的運移行為。
在這份論文中，無論是程式語言或是數值模擬方法，都是以前我沒有學過的技能，而碩士班的訓練又跟過去的求學階段不同，不會再有人把答案整理好給你，老師只會給一個大概的方向，要靠自己摸索、不斷嘗試，去釐清問題、尋找方法、製造工具，再解決問題，歸納出屬於自己的答案。
在研究的過程中我遇到許多困難，現在回頭想，那段時間我經歷了許多次崩潰後又再整理好自己繼續努力下去的時刻，面對不停出現的困難，我明顯地感受到自己正處在負面情緒當中，但我沒有選擇放棄或逃避，我透過修課、自學，找老師討論等方式，不斷嘗試各種方法去解決問題，這條路不通就再換一條，每天往前進一點點，即使是只前進一小步也無妨，再氣餒我都不放棄，然後就慢慢完成論文，走完這段過程了。

關於這份申請網頁，請分享一個你開發時的技術心得。

我發現CSS真的是一門要專精很不容易的領域!!有各式各樣讓網頁變得更華麗、美觀的指令和設計技巧可以學習!!
另外切版也是一門重要的學問，需要思考該如何讓網頁在不同裝置上都能不爆版且容易閱讀。
此外，撰寫這份網頁時，我嘗試站在使用者的角度思考，該如何讓網頁的排版和呈現更容易閱讀，是很特別的體驗!

其他想要對我們說的事情？

謝謝你們用心開設這個課程，一個人在轉職的路上很容易感到徬徨，常常會覺得不曉得該何去何從，好像學了各式各樣的東西，東拼西湊的感覺沒有系統性，總有種不踏實的感覺。
決定要報名WeHelp課程前，做了不少功課，研究了各種坊間的訓練班，發現WeHelp課程安排充足的時間讓我去獨立完成專案，只給有限的指導，能夠訓練我獨立解決問題的能力。
而且WeHelp求職導向的模式，也很符合我現階段的需求，感覺像是有了一個方向，有人可以指引我完成轉職的準備，雖然轉職的路還是要靠我自己一步一步踏實地走完，但有一套完整的學習方向，還是能讓人感到踏實許多。
由衷地感謝你們的存在!
